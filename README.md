# Система разделения музыки на инструментальные дорожки
Этот проект представляет собой инструмент для автоматического разделения аудиозаписей на четыре компонента (стэма): вокал, барабаны, бас и прочие инструменты.

### Структура проекта
music-separation/
├── analysis/         # Ноутбук с анализом полученных результатов
├── samples/          # Аудиофайл, используемый для тестирования инференса
├── scripts/          # Основные скрипты:
│   ├── separate_single.py  # Инференс для одного файла
│   ├── evaluate.py         # Оценка качества модели на датасете MUSDB18
│   ├── metrics.png         # Визуализация метрик качества (SDR, SIR, SAR) по результатам тестирования
│   └── app.py              # FastAPI веб-сервис для разделения аудио
├── requirements.txt
└── report.txt        # Отчет о проделанной работе

### О проекте
* Использование модели htdemucs для качественной декомпозиции аудио.
* Скрипт evaluate.py вычисляет метрики SDR, SIR, SAR (используя museval) на тестовой выборке MUSDB18.
* Веб-сервис на FastAPI, который принимает аудиофайл и возвращает ZIP-архив с выделенными стэмами.

### Установка
Клонируйте репозиторий:
```
git clone https://github.com/nadfadeeva/music-separation.git
cd music-separation
```
Установите зависимости:
```
pip install -r requirements.txt
```

### Использование
1. Разделение отдельного аудиофайла
Для обработки файла запустите скрипт:
```
python scripts/separate_single.py path/to/your/song.mp3
```
Результаты будут сохранены в папку scripts/separated/.

2. Оценка качества модели
Скрипт проводит оценку на датасете MUSDB18 и сохраняет график метрик (metrics.png):
```
python scripts/evaluate.py
```
3. Запуск веб-сервиса
Запустите API для удаленной работы:
```
python scripts/app.py
```
Протестировать: http://127.0.0.1:8000/docs

### Ключевые выводы
* Модель отлично справляется с вокалом, голос почти всегда получается чистым и качественным.
* Сложнее всего работать с басом и ударными. Если в песне они звучат близко друг к другу или перекрываются, модель начинает путаться.
* Категория "Other" работает хуже остального, в неё попадает всё, что модель не смогла распознать.
* Минус - низкая скорость обработки, которая длится около 5 минут на один трек.
